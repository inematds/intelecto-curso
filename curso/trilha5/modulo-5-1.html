Módulo 5.1

Os 6 Padrões de Arquitetura Claw

OpenClaw, ZeroClaw, NanoClaw, TinyClaw, IronClaw e Agent Zero — quando e por que usar cada padrão.

1

O que é uma Arquitetura Claw?

O nome "Claw" (garra) representa a capacidade de cada padrão de agarrar um conjunto específico de trade-offs entre velocidade, segurança, autonomia e custo.

Os 6 Padrões Claw
Padrão Foco Complexidade Caso de Uso
OpenClawMáxima produtividadeBaixaUso pessoal irrestrito
ZeroClawZero dependênciasBaixaOffline, privacidade total
NanoClawMinimalismo extremoMínimaEdge, IoT, Raspberry Pi

2

Por que Aprender os 6 Padrões?

Conhecer apenas um padrão é como ter somente um martelo. Cada Claw resolve um conjunto diferente de problemas, e escolher errado pode significar semanas de retrabalho.

Vantagens de conhecer todos
  • · Escolha fundamentada para cada projeto
  • · Migracao suave entre padrões
  • · Reutilizacao de componentes entre Claws
Custos de escolher errado
  • · IronClaw em prototipo = lentidão desnecessária
  • · OpenClaw em producao = vulnerabilidade crítica
  • · NanoClaw para multi-agentes = impossível escalar
3

OpenClaw — O Assistente Aberto

OpenClaw é o padrão padrao do INTELECTO — acesso completo ao sistema de arquivos, internet e ferramentas. Ideal para uso pessoal onde você é o único usuário.

# OpenClaw config (intelecto.yaml)
architecture: openclaw
security:
  sandbox: false
  file_access: unrestricted
  network: unrestricted
tools:
  auto_discover: true
  require_approval: false
4

ZeroClaw e NanoClaw — Leveza Máxima

ZeroClaw elimina todas as dependências externas — funciona 100% offline. NanoClaw vai além: um único arquivo Python com menos de 500 linhas.

ZeroClaw
  • Modelo: Ollama local (Llama 3, Mistral)
  • Memória: SQLite local
  • Canais: Terminal ou interface web local
NanoClaw
  • Tamanho: menos de 500 linhas Python
  • Dependências: Apenas stdlib + uma lib de LLM
  • Alvo: Raspberry Pi, VPS mínimo
5

IronClaw — Máxima Segurança

IronClaw implementa Zero-Trust completo: cada operacao é auditada, ferramentas executam em sandbox isolado, e nenhuma ação ocorre sem aprovação explícita para operações de alto risco.

Stack de Segurança IronClaw

4Aprovação humana para operações destrutivas

6

Agent Zero — Autoconstrucao e Meta-Cognição

O padrão mais avançado: agentes que analisam seu próprio desempenho, identificam lacunas e criam ou modificam suas próprias ferramentas e memórias para melhorar progressivamente.

Loop de Auto-Evolucao
Executar → Observar → Avaliar → Melhorar → Persistir
Salvaguardas obrigatórias
  • · Rate limit de auto-modificacoes por período
  • · Aprovacao humana antes de alteracoes em SOUL.md
  • · Rollback automático se métricas piorarem
